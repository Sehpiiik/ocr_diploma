# Noise generator

[English](./README.md) | [Русский](./README_RU.md)

Применяет реалистичные **фотометрические** искажения сканера / копировального аппарата к чистым
изображениям документов **на месте**. Размеры изображения и входные аннотации bbox
сохраняются точно - без поворота, без перспективных искажений.

Процесс применяется выборочно для каждого документа: обрабатываются только те пары, в чьей JSON-аннотации
есть `"augment": true`.

## Входные данные

```
<input_dir>/
  images/       doc_00001.png, doc_00002.png, ...
  annotations/  doc_00001.json, doc_00002.json, ...
```

Каждая аннотация использует схему, созданную `documents_generator`. Чтобы отметить документ
для аугментации, нужно установить флаг `augment` в `true`:

```json
{
  "image": "doc_00001.png",
  "width": 2480,
  "height": 3508,
  "bbox_format": "xywh",
  "augment": true,
  "objects": [
    {"text": "...", "bbox": [x, y, w, h], "type": "line"}
  ]
}
```

Пары без `"augment": true` (отсутствующий ключ, `false` и т.д.) пропускаются.

## Поведение

Для каждой отмеченной пары генератор:

1. Записывает `--variants` аугментированных копий в те же **самые** папки `images/` и
   `annotations/`, именуя их с суффиксом `_au_NN`:

   ```
   <input_dir>/
     images/       doc_00001_au_01.png, doc_00001_au_02.png, ...
     annotations/  doc_00001_au_01.json, doc_00001_au_02.json, ...
   ```

2. Каждая новая аннотация является дословной копией исходной аннотации, в которой
   поле `image` перезаписано на новое имя файла варианта.

3. После успешной записи всех вариантов для этой пары **оригинальные**
   изображение и аннотация удаляются.

Если какая-либо запись не удалась, оригиналы остаются на месте, чтобы запуск можно было
повторить. `--keep-originals` полностью отключает шаг удаления.

## Применяемые искажения

Для каждого изображения срабатывает случайное подмножество эффектов (каждый эффект имеет свою
вероятность, порядок перемешивается для каждого изображения):

1. **Бумага и чернила** (augraphy):
   - `InkBleed`, `LowInkRandomLines`, `Letterpress`
   - `BleedThrough` (содержимое, просвечивающее с обратной стороны)
2. **Артефакты копировального аппарата / сканера** (augraphy):
   - `BadPhotoCopy` — выцветшие области, полосы памяти
   - `DirtyDrum`, `DirtyRollers` — вертикальные/горизонтальные тональные полосы
   - `LightingGradient`, `ShadowCast` — неравномерное освещение
   - `BrightnessTexturize`, `SubtleNoise`
3. **Пользовательские сенсорные / оптические эффекты** (OpenCV + NumPy):
   - Виньетирование (радиальное затемнение)
   - Блик сканера (яркое пятно)
   - Пыль и царапины
   - Горизонтальные полосы сканирования
   - Размытие в движении (лёгкое размазывание сканера)
   - Передискретизация с низким DPI (понижение + повышение разрешения, размеры сохраняются)
   - Гауссов шум сенсора
   - JPEG-сжатие

Все эффекты затрагивают только пиксели - они не изменяют размеры изображения, поэтому
координаты bbox остаются корректными для аугментированных вариантов.

## Установка

```bash
pip install -r requirements.txt
```

## Использование

```bash
python noise_generator.py \
    --input ../documents_generator/out \
    --variants 3 \
    --seed 42
```

| Флаг | Описание |
|------|-------------|
| `--input DIR` | Директория, содержащая `images/` + `annotations/` (обязательно). Аугментированные файлы записываются обратно в эти папки. |
| `--variants N` | Количество аугментированных вариантов для каждой отмеченной пары (по умолчанию: 1). Индекс варианта дополняется нулями в суффиксе `_au_NN`. |
| `--seed INT` | Воспроизводимый seed RNG. |
| `--preset {light,medium,heavy,scanner,photocopy}` | Интенсивность / вариант. |
| `--debug-boxes` | Рисовать рамки xywh на каждом сохранённом аугментированном изображении. |
| `--limit N` | Обработать только первые N отмеченных пар. |
| `--workers N` | Параллельные рабочие процессы (по умолчанию: 1). |
| `--keep-originals` | Не удалять исходную пару после записи вариантов. |

## Пример

```bash
# Генерация чистых пар и определение, какие из них аугментировать...
cd ../documents_generator && python doc_generator.py --corpus sample_corpus.txt -n 5
# (действие / ручной шаг, устанавливающий "augment": true в нужных JSON)

# ...затем применение шумов к отмеченным парам на месте.
cd ../noise_generator
python noise_generator.py --input ../documents_generator/out --variants 2 --seed 0
```

После выполнения исходная отмеченная пара `doc_00001.{png,json}` удаляется и
заменяется на `doc_00001_au_01.{png,json}` и `doc_00001_au_02.{png,json}`.
